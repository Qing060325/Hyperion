## 🤝 贡献者指南

### 开发流程
1. Fork 仓库并创建特性分支
2. 编写功能和对应的单元测试
3. 遵循 ESLint 和 Prettier 规范
4. 提交 PR 并通过代码审查
5. 通过 CI/CD 流程自动测试和部署

### 代码规范
- 使用 TypeScript 严格模式
- 遵循 SolidJS 最佳实践
- 保持组件小而专注
- 复用通用 hooks 和工具函数
- 编写有意义的提交信息

### 测试要求
- 新功能必须有单元测试
- 复杂逻辑需要集成测试
- UI 变更需要视觉回归测试
- 性能变更需要基准测试
