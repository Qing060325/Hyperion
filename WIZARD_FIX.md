# 向导遮挡问题修复说明

## 问题描述
用户报告"很多按钮按了没反应"，经诊断发现是欢迎向导覆盖层遮挡了整个页面。

## 根本原因
1. **默认设置问题**: `wizard_completed` 默认值为 `false`，导致首次运行时向导覆盖整个页面
2. **向导覆盖层**: `WelcomeWizard` 组件使用 `fixed inset-0 z-50` 覆盖整个屏幕
3. **状态持久化问题**: 即使用户点击"跳过向导"，状态可能没有正确保存到 localStorage

## 修复内容

### 1. 修改默认设置 (`src/types/clash.ts`)
```typescript
// Line 268
wizard_completed: true,  // 改为 true，新用户不会看到向导
```

### 2. 添加智能检测逻辑 (`src/App.tsx`)
- 检查 localStorage 中是否有 Clash 连接配置
- 如果检测到已有配置，自动跳过向导
- 添加控制台调试命令

### 3. 添加关闭按钮 (`src/components/wizard/WelcomeWizard.tsx`)
- 在向导右上角添加 X 关闭按钮
- 用户可以随时关闭向导

### 4. 改进状态保存 (`src/stores/settings.ts`)
- 添加详细的控制台日志
- 改进错误处理

## 控制台调试命令

在浏览器控制台中可以使用以下命令：

```javascript
// 关闭向导
window.closeWizard()

// 重置向导（下次运行时显示）
window.resetWizard()

// 立即显示向导
window.showWizard()
```
